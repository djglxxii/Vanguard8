# Showcase Milestone 3 — Tile-Mode and Sprite Verification Scenes

Objective:
- Add the tile-mode and sprite scenes that verify Graphic 1/2 usage and the
  covered sprite behaviors needed for title/menu-style ROM testing.

Required docs:
- `docs/spec/02-video.md`
- `docs/spec/04-io.md`
- `showcase/docs/showcase-rom-plan.md`
- `showcase/docs/showcase-implementation-plan.md`

Allowed scope:
- Tile-mode presentation scene, sprite capability scene, generated source
  assets for those scenes, and related checkpoint metadata.

Explicit non-goals:
- Mixed-mode or command-engine scenes.
- Any speculative sprite behavior outside the covered emulator surface.

Allowed paths:
- `showcase/`

Required verification:
- Graphic 1/2 scene output is stable and visually interpretable.
- Sprite size, magnification, overlap, and limit behavior are visible in a
  deterministic scene.
- The scene checkpoints are documented for human review.

Exit criteria:
- A reviewer can inspect tile-mode and sprite scene captures and tell whether
  the intended hardware behavior is present.

Forbidden extras:
- Any dependency on unverified text-mode or scroll behavior.

Verification commands:
```bash
python3 showcase/tools/package/build_showcase.py
python3 showcase/tools/package/run_showcase_headless.py --frames 720 --trace build/showcase/m3.trace --hash-frame 719 --symbols build/showcase/showcase.sym
```
