# Milestone 9 — Debugger Foundation

Objective:
- Add the core tooling needed to inspect timing, mapping, and rendering state.

Required docs:
- `docs/emulator/06-debugger.md`
- `docs/emulator/07-implementation-plan.md`

Allowed scope:
- ImGui shell, docking layout, CPU panel, memory panel, VDP panel, interrupt
  log, bank tracker, and basic breakpoints/watchpoints.
- Read-only core inspection accessors and minimal debugger control entrypoints
  needed to surface those panels without ad hoc logging.

Explicit non-goals:
- Save states, rewind, replay, or extra debugger panels beyond the foundation.

Allowed paths:
- `src/debugger/`
- `src/frontend/`
- `src/core/`
- `src/core/cpu/`
- `third_party/z180/`
- `tests/`

Required tests:
- Debugger attachment does not mutate emulation state.
- Run/pause/step stop at correct boundaries.
- Breakpoints trigger deterministically.

Exit criteria:
- Core emulator issues can be diagnosed without ad hoc runtime logging.

Forbidden extras:
- Speculative tools that depend on milestone 10 state snapshots.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
