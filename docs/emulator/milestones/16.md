# Milestone 16 — Graphic 6 Renderer and Mixed-Mode HUD Compatibility

Objective:
- Add the narrow `Graphic 6` (`Screen 7`) rendering and mixed-mode coverage
  needed for a fixed `512x212` HUD overlay composited over a scrolling
  `Graphic 4` background in a future showcase scene.

Required docs:
- `docs/spec/02-video.md`
- `docs/spec/04-io.md`
- `docs/emulator/04-video.md`
- `docs/emulator/07-implementation-plan.md`
- `docs/emulator/08-compatibility-audit.md`
- `docs/emulator/09-1.0-readiness.md`

Allowed scope:
- `Graphic 6` background rendering from the documented mode bits and pixel
  packing
- Narrow mixed-mode compositor/runtime coverage for `VDP-A Graphic 6` over
  `VDP-B Graphic 4`
- The minimal display upload/readback seam needed to carry a composed
  `512x212` RGB frame
- Docs and tests needed to lock the supported `Graphic 6` surface

Explicit non-goals:
- `Graphic 5` palette mapping work
- `Graphic 7` direct-color work
- `Graphic 5/6/7` command-engine packing
- New showcase-ROM scene implementation
- General V9938 mode-completion or frontend/debugger work

Allowed paths:
- `src/core/`
- `src/frontend/`
- `tests/`
- `docs/`

Required tests:
- `Graphic 6` framebuffer-addressing coverage for the documented
  `byte_addr = (y * 256) + (x / 2)` packing
- Palette/output regression coverage proving the two-nibbles-per-byte path
  renders deterministically at `512x212`
- Mixed-mode compositor/runtime coverage for `VDP-A Graphic 6` over
  `VDP-B Graphic 4`
- Display upload/readback coverage for the new `512x212` frame size

Exit criteria:
- Unsupported `Graphic 6` usage no longer falls back to backdrop output
- The emulator can render deterministic `Graphic 6` frames suitable for a ROM
  HUD overlay without introducing mode-specific shortcuts
- The remaining `Graphic 5`, `Graphic 7`, and `Graphic 5/6/7` command-packing
  gaps remain documented rather than partially implemented

Forbidden extras:
- Turning the milestone into a broader high-resolution mode pass
- Implementing `Text 1`/`Text 2`, `Graphic 5`, or `Graphic 7` under the same
  milestone
- Extending horizontal scroll behavior beyond the already documented `R#23`
  vertical-scroll path needed by the background layer

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
