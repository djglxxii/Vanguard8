# Milestone 31 — Timed HD64180 PacManV8 ROM Run-Time Opcode Coverage

Objective:
- After M30 closed the VBlank handler prologue/epilogue, running the full
  PacManV8 `pacman.rom` against the timed HD64180 dispatcher still aborts on
  a series of general-purpose opcodes used by the main program past the
  interrupt boundary. The user has explicitly broadened this milestone
  (deviating from the M19-M30 one-opcode-at-a-time discipline) to cover all
  timed-HD64180 opcode gaps discoverable by running the canonical
  `cmake-build-debug/src/vanguard8_headless --rom pacman.rom --frames 3600
  --hash-audio` binary against the current PacManV8 `pacman.rom`, so that
  downstream bring-up work does not continue to hit a rebuild-per-opcode
  repair loop.
- The discovered opcode gaps, in the order they surfaced during iterative
  discovery, were:
  - `0xC0` (`RET NZ`)
  - `0xCA` (`JP Z, nn`), plus the sister `0xC2` (`JP NZ, nn`)
  - `0x28` (`JR Z, e`)
  - `0x57` (`LD D, A`) — symptomatic of the missing general `LD r, r'`
    family beyond the `LD A, r` forms covered by M27
  - `0x3D` (`DEC A`) — symptomatic of the missing general `DEC r` / `INC r`
    family beyond the `DEC B` / `INC HL` / `DEC DE` singletons
  - `0xFE` (`CP n`)

Allowed scope:
- Timed opcode coverage inside the existing extracted HD64180 runtime for
  the opcode families below, using the authoritative Z80/HD64180 semantics:
  - `LD r, r'` / `LD r, (HL)` / `LD (HL), r` for the entire `0x40-0x7F`
    block except `0x76` (`HALT`). This includes register-to-register
    moves for `B/C/D/E/H/L/A` destinations and sources, and the
    `(HL)`-indirect load/store forms.
  - `INC r` / `DEC r` for `B/C/D/E/H/L/A` (opcodes `0x04/0x0C/0x14/0x1C/
    0x24/0x2C/0x3C` for INC and `0x05/0x0D/0x15/0x1D/0x25/0x2D/0x3D`
    for DEC), plus the `(HL)` indirect forms `0x34` (`INC (HL)`) and
    `0x35` (`DEC (HL)`), with correct S/Z/H/P/V/N flag semantics
    (carry preserved; overflow on 0x7F->0x80 for INC and 0x80->0x7F
    for DEC).
  - `JR Z, e` (`0x28`), mirroring the existing `JR NZ, e` semantics.
  - `JP NZ, nn` (`0xC2`) and `JP Z, nn` (`0xCA`), mirroring the
    existing `JP nn` semantics with the appropriate flag gating.
  - `RET NZ` (`0xC0`), mirroring the existing `RET Z` semantics.
  - `CP n` (`0xFE`) and the register/indirect `CP r` / `CP (HL)` family
    `0xB8-0xBF`, with correct S/Z/H/P/V/N/C flag semantics and no write
    to `A`.
- Adapter T-state timing entries for every opcode added in this milestone,
  matching the Z80/HD64180 data-book cycle counts (4 T-states for
  register-to-register and `INC r`/`DEC r` register forms; 7 T-states for
  `LD r, (HL)` / `LD (HL), r` / `CP (HL)` / `CP n`; 10 T-states for
  `JP cc, nn`; 11/5 T-states for `RET NZ` taken/not-taken; 11 T-states
  for `INC (HL)` / `DEC (HL)`; 12/7 T-states for `JR Z, e` taken/not-taken).
- Tests and docs needed to lock the newly covered timed opcode surface.
- Runtime/integration coverage demonstrating the full PacManV8 `pacman.rom`
  runs against the timed dispatcher for at least the T016 regression
  window without a timed-opcode abort.

Explicit non-goals:
- Full Z80/Z180 opcode completion. This milestone covers only the
  opcode families made necessary by the PacManV8 ROM, as surfaced by the
  iterative discovery loop documented in the
  `vanguard8-build-directory-skew-and-timed-opcodes` field manual.
- Index register (`IX`/`IY`) opcodes or `DD`/`FD` prefix support.
- `CB`-prefixed bit/shift/rotate opcodes beyond what is already supported.
- Block-move and block-compare instructions (`LDIR`, `LDDR`, `CPIR`, etc.)
  beyond those already supported.
- Arithmetic ops beyond `INC`/`DEC`/`CP` (no new `ADD`/`SUB`/`ADC`/`SBC`
  families in this milestone). If the ROM surfaces those later, they belong
  to a future milestone.
- Expanding the `ED`-prefix coverage past what is already implemented.
- VDP, palette, renderer, scheduler, audio, or frontend changes unrelated
  to the timed CPU opcode gaps.

Allowed paths:
- `src/core/`
- `third_party/z180/`
- `tests/`
- `docs/`

Required tests:
- Focused timed CPU coverage for a representative sample of the
  `LD r, r'` dispatcher: at minimum one `LD r, r'` case per destination
  register (`B`, `C`, `D`, `E`, `H`, `L`, `A`), plus one `LD r, (HL)`
  case and one `LD (HL), r` case, pinning the documented semantics
  (value copy, 4 T-states for register-to-register, 7 T-states for
  `(HL)` indirect, flags untouched).
- Focused timed CPU coverage for `INC r` / `DEC r` that pins:
  - Half-carry set when the low nibble crosses `0x0F` (INC) or
    `0x00` (DEC).
  - Parity/overflow flag set on the signed-overflow boundary
    (`0x7F->0x80` for INC, `0x80->0x7F` for DEC).
  - Carry flag preserved across `INC` / `DEC`.
  - 4 T-states for register forms; 11 T-states for `(HL)` forms.
- Focused timed CPU coverage for `JR Z, e`, `JP Z, nn`, `JP NZ, nn`,
  `RET NZ` that pins the taken / not-taken cycle counts and PC/SP
  behavior against the zero-flag state.
- Focused timed CPU coverage for `CP n` / `CP r` / `CP (HL)` pinning:
  - `A` is not modified.
  - Z=1 when `A == operand`, C=1 when `A < operand`, N=1 always.
  - Half-borrow flagged when low nibble of `operand` exceeds low
    nibble of `A`.
  - 7 T-states for `CP n` and `CP (HL)`; 4 T-states for `CP r`.
- Runtime/integration coverage that drives the full PacManV8
  `pacman.rom` through the timed dispatcher for at least 3600
  emulated frames without a timed-opcode abort, verifying that the
  audio SHA-256 and event-log digest remain stable across repeat runs
  (determinism proof for the extended coverage).

Exit criteria:
- The canonical `cmake-build-debug/src/vanguard8_headless --rom
  /path/to/pacman.rom --frames 3600 --hash-audio` command runs to
  completion without a timed-opcode abort.
- All Catch2 `ctest` suites pass against the updated dispatcher.
- The added opcode families are narrow, documented, and regression-covered.
- Any further opcode gaps surfaced by longer runs or new ROM code belong
  to a subsequent narrow compatibility milestone; this milestone does not
  retroactively expand to absorb them.

Forbidden extras:
- Turning this milestone into an index-register or `CB`-prefix rewrite.
- Bundling unrelated PacManV8 compatibility fixes under the same task.
- Loosening existing timed CPU or integration tests instead of adding the
  missing opcode coverage.
- Adding VDP, palette, audio, or frontend changes to this milestone.

Verification commands:
```bash
cmake --build cmake-build-debug
ctest --test-dir cmake-build-debug --output-on-failure
cmake-build-debug/src/vanguard8_headless \
    --rom /home/djglxxii/src/PacManV8/build/pacman.rom \
    --frames 3600 --hash-audio
```
