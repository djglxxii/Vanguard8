# Milestone 21 — Timed HD64180 Boot Opcode Compatibility (PacManV8)

Objective:
- Close the narrow timed-CPU compatibility gap exposed by the PacManV8 minimal
  boot ROM: the extracted timed HD64180 runtime aborts on `LD C,n` (`0x0E`)
  and `LD D,n` (`0x16`) during the boot/init path.

Required docs:
- `docs/spec/01-cpu.md`
- `docs/spec/04-io.md`
- `docs/emulator/03-cpu-and-bus.md`
- `docs/emulator/07-implementation-plan.md`
- `docs/emulator/15-second-pass-audit.md`

Allowed scope:
- Timed opcode coverage inside the existing `src/core/cpu/` extracted-HD64180
  runtime path
- Tests and docs needed to lock the newly covered timed opcode surface
- Narrow runtime/integration coverage for the observed PacManV8 boot blocker

Explicit non-goals:
- Full Z80/Z180 opcode completion
- New index-register, CB-prefixed, or block-instruction work unless the active
  failing path proves it is required
- Frontend, renderer, scheduler, or audio changes unrelated to the timed CPU
  blocker
- New ROM content or PacManV8-side work bundled into the same milestone

Allowed paths:
- `src/core/`
- `tests/`
- `docs/`

Required tests:
- Focused timed CPU coverage for `LD C,n` (`0x0E`)
- Focused timed CPU coverage for `LD D,n` (`0x16`)
- Runtime/integration coverage reproducing the PacManV8 boot blocker and
  proving execution survives the previously failing PC `0x012A`

Exit criteria:
- The emulator no longer aborts on the documented PacManV8 boot path because
  timed execution is missing opcodes `0x0E` or `0x16`
- The added timed opcode coverage is narrow, documented, and regression-covered
- Remaining opcode gaps stay explicit instead of expanding this milestone into
  a general CPU-completion pass

Forbidden extras:
- Turning the milestone into a broad CPU rewrite
- Bundling unrelated real-ROM compatibility fixes under the same task
- Loosening existing timed CPU or integration tests instead of adding the
  missing opcode coverage

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
