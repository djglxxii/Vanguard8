# Milestone 19 — Timed HD64180 Boot Opcode Compatibility

Objective:
- Close the narrow timed-CPU compatibility gap exposed by the Pac-Man
  boot/background bring-up evidence: the extracted timed HD64180 runtime still
  throws on `LD B,n` (`0x06`) and `IM 1` (`ED 0x56`).

Required docs:
- `docs/spec/01-cpu.md`
- `docs/spec/04-io.md`
- `docs/emulator/03-cpu-and-bus.md`
- `docs/emulator/07-implementation-plan.md`
- `docs/emulator/15-second-pass-audit.md`

Allowed scope:
- Timed opcode coverage inside the existing `src/core/cpu/` extracted-HD64180
  runtime path
- Tests and docs needed to lock the newly covered timed opcode surface
- Narrow runtime/integration coverage for the observed Pac-Man
  boot/background blocker

Explicit non-goals:
- Full Z80/Z180 opcode completion
- New index-register, CB-prefixed, or block-instruction work unless the active
  failing path proves it is required
- Frontend, renderer, scheduler, or audio changes unrelated to the timed CPU
  blocker
- New ROM content or showcase-side work bundled into the same milestone

Allowed paths:
- `src/core/`
- `tests/`
- `docs/`

Required tests:
- Focused timed CPU coverage for `LD B,n`
- Focused timed CPU coverage for `IM 1`
- Coverage proving `IM 1` does not violate the documented mode-independent
  `INT1` behavior
- Runtime/integration coverage reproducing the Pac-Man boot/background blocker
  and proving execution survives the previously failing PCs `0x0140` and
  `0x0314`

Exit criteria:
- The emulator no longer aborts on the documented Pac-Man boot/background path
  because timed execution is missing `0x06` or `ED 0x56`
- The added opcode coverage is narrow, documented, and regression-covered
- Remaining opcode gaps stay explicit instead of expanding this milestone into
  a general CPU-completion pass

Forbidden extras:
- Turning the milestone into a broad CPU rewrite
- Bundling unrelated real-ROM compatibility fixes under the same task
- Loosening existing interrupt-model tests instead of adding the missing timed
  opcode coverage

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
