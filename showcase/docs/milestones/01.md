# Showcase Milestone 1 — Bring-Up Cartridge, Scene Loop, and First Checkpoint

Objective:
- Prove ROM-side boot/MMU/vector flow, scene sequencing, and the first banked
  checkpoint using deterministic timing and symbol-aware traceability.

Required docs:
- `docs/spec/00-overview.md`
- `docs/spec/01-cpu.md`
- `docs/spec/04-io.md`
- `showcase/docs/showcase-rom-plan.md`
- `showcase/docs/showcase-implementation-plan.md`

Allowed scope:
- Boot code, scene-loop scaffolding, banked ROM layout, checkpoint metadata,
  and the docs required to review those flows.

Explicit non-goals:
- Final visual identity, audio scenes, or later video-scene content.
- Any broad DMA usage beyond a narrow documented need that arises in review.

Allowed paths:
- `showcase/`

Required verification:
- The ROM boots through its scene loop deterministically.
- At least one banked transition is visible in trace and/or frame output.
- The first named checkpoint can be reproduced from the documented run command.

Exit criteria:
- A reviewer can build the ROM, run the first loop, and inspect a stable trace
  or frame result for the first checkpoint.

Forbidden extras:
- Speculative scene systems or data abstractions aimed at later milestones.

Verification commands:
```bash
python3 showcase/tools/package/build_showcase.py
python3 showcase/tools/package/run_showcase_headless.py --frames 240 --trace build/showcase/m1.trace --hash-frame 239 --symbols build/showcase/showcase.sym
```
