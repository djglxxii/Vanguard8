# Milestone 12 — Playable Desktop Frontend Baseline

Objective:
- Replace the current terminal-only frontend launcher with a real desktop
  application that can launch a ROM, show live video, play live audio, accept
  basic input, and expose lightweight bring-up/debug status.

Required docs:
- `docs/spec/04-io.md`
- `docs/emulator/00-overview.md`
- `docs/emulator/01-build.md`
- `docs/emulator/04-video.md`
- `docs/emulator/05-audio.md`
- `docs/emulator/07-implementation-plan.md`
- `docs/emulator/10-desktop-gui-audit.md`

Allowed scope:
- SDL window lifecycle, OpenGL context creation, persistent frontend runtime
  loop, real frame presentation, backend-neutral presentation/readback seams,
  command-line ROM launch into the live desktop session, SDL audio output,
  minimal live input handling, and narrow runtime status/error/debug output.

Explicit non-goals:
- Drag-and-drop/native file dialog ROM UX, recent-ROM polish, or other desktop
  workflow expansion beyond command-line ROM launch.
- Dear ImGui debugger rendering.
- Save-state, rewind, quicksave/quickload, replay, screenshot, or WAV-export
  UX work.
- New emulator-core compatibility work unrelated to frontend hosting.

Allowed paths:
- `src/frontend/`
- `src/core/config.*`
- `src/core/emulator.*`
- `src/core/audio/`
- `src/main.cpp`
- `src/CMakeLists.txt`
- `tests/`
- `docs/`

Required tests:
- Frontend runtime lifecycle coverage using fake window/display/audio hosts.
- Presentation/readback coverage for a known frame through the desktop display
  path.
- Audio device queue/ring buffer coverage for startup, shutdown, and covered
  underrun behavior.
- Minimal SDL event mapping coverage through frontend host abstractions.
- Backend initialization, ROM-load failure, and clean quit handling.

Exit criteria:
- Launching `vanguard8_frontend --rom <path>` opens a visible window, remains
  open until the user exits, displays live ROM-driven frames, and produces
  audible audio.
- The frontend can present a fixture frame or ROM-driven frame through a real
  presentation backend without regressing deterministic headless output.
- Covered live input reaches the controller path and lightweight runtime/debug
  output is available without a full desktop debugger.

Forbidden extras:
- Implementing the heavy compatibility backlog from
  `docs/emulator/11-pre-gui-audit.md` inside milestone 12.
- Full desktop debugger or Dear ImGui panel rendering.
- Desktop workflow polish beyond the narrow playable-baseline goal.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
