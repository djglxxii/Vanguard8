# Milestone 20 — Timed HD64180 Palette VCLK Compatibility

Objective:
- Close the narrow timed-CPU compatibility gap exposed by the Pac-Man palette
  swatch evidence: the runtime succeeds with `VCLK: off` but the `VCLK 4000`
  path still throws on timed opcode `0x05` at `PC 0x02B9`.

Required docs:
- `docs/spec/01-cpu.md`
- `docs/spec/03-audio.md`
- `docs/spec/04-io.md`
- `docs/emulator/03-cpu-and-bus.md`
- `docs/emulator/07-implementation-plan.md`
- `docs/emulator/15-second-pass-audit.md`

Allowed scope:
- Timed opcode coverage inside the existing `src/core/cpu/` extracted-HD64180
  runtime path
- Tests and docs needed to lock the newly covered timed opcode surface
- Narrow runtime/integration coverage for the observed Pac-Man palette swatch
  blocker under active `VCLK`

Explicit non-goals:
- Broad palette remapping or VDP palette-surface changes
- New VDP command-engine behavior unless the active failing path proves it is
  required
- Full Z80/Z180 opcode completion
- Frontend or ROM-side work bundled into the same milestone

Allowed paths:
- `src/core/`
- `tests/`
- `docs/`

Required tests:
- Focused timed CPU coverage for opcode `0x05`
- Runtime/integration coverage reproducing the Pac-Man palette swatch blocker
  and proving execution survives the previously failing `PC 0x02B9` with
  `VCLK 4000`
- Coverage proving the already-passing `VCLK: off` palette path remains stable

Exit criteria:
- The emulator no longer aborts on the documented Pac-Man palette swatch path
  because timed execution is missing opcode `0x05`
- The added timed opcode coverage is narrow, documented, and regression-covered
- Remaining opcode gaps stay explicit instead of expanding this milestone into
  a general CPU-completion pass

Forbidden extras:
- Turning the milestone into a broad VDP or palette rewrite
- Bundling unrelated real-ROM compatibility fixes under the same task
- Loosening existing timed CPU or integration tests instead of adding the
  missing opcode coverage

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
