# Showcase Milestone 0 — Showcase Workspace Bootstrap and Stable Build Entry Points

Objective:
- Create the ROM-side source tree, stable build/run wrappers, and a minimal
  bootable ROM image that proves the showcase workspace can produce
  `showcase.rom` and `showcase.sym`.

Required docs:
- `docs/spec/00-overview.md`
- `docs/spec/01-cpu.md`
- `docs/emulator/02-emulation-loop.md`
- `showcase/docs/showcase-rom-plan.md`
- `showcase/docs/showcase-implementation-plan.md`

Allowed scope:
- Showcase ROM scaffolding, build/run wrappers, ROM entrypoint, and the minimum
  docs needed to make those interfaces reviewable.

Explicit non-goals:
- Multi-scene content, final checkpoint manifests, or polished assets.
- Audio, mixed-mode, or compositing verification beyond what is required to
  prove the scaffold works.

Allowed paths:
- `showcase/`

Required verification:
- Build wrapper emits a ROM and symbol file deterministically.
- Headless-run wrapper can launch the ROM for a fixed frame count.
- The ROM reaches an identifiable screen state suitable for manual smoke review.

Exit criteria:
- One documented build command produces `showcase.rom` and `showcase.sym`.
- One documented run command can produce at least one trace or frame artifact.

Forbidden extras:
- Any expansion of emulator-core behavior or root-level workflow unrelated to
  the showcase workspace.

Verification commands:
```bash
python3 showcase/tools/package/build_showcase.py
python3 showcase/tools/package/run_showcase_headless.py --frames 1 --trace build/showcase/m0.trace
```
