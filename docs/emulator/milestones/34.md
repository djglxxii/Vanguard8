# Milestone 34 — Timed HD64180 PacManV8 Intermission Opcode Coverage

Objective:
- Close the next real-ROM timed CPU compatibility gap exposed after M33 by the
  PacManV8 blocked task
  `/home/djglxxii/src/PacManV8/docs/tasks/blocked/T020-intermission-cutscenes.md`.
  Wiring the T020 intermission owner into the current PacManV8 ROM now reaches
  a small cluster of missing timed opcodes on the level-progression and
  intermission update/render path. The confirmed missing opcodes are:
  - `ED 52` (`SBC HL,DE`) at `PC 0x089E`
  - `0x10` (`DJNZ`) at `PC 0x32D3`
  - `0x87` (`ADD A,A`) at `PC 0x32E9`
  - `0x30` (`JR NC,e`) at `PC 0x2F45`
  - `0x09` (`ADD HL,BC`) at `PC 0x333F`
- The same blocker record also identifies directly adjacent sister opcodes on
  the same active path:
  - `0x38` (`JR C,e`) immediately after the confirmed `JR NC` site
  - `0x19` (`ADD HL,DE`) in the same row-stride and table-walk patterns
- This milestone exists to close that whole blocker set in one documented pass
  so PacManV8 T020 can resume without falling back into another
  rebuild-per-opcode loop.

Allowed scope:
- Timed opcode coverage inside the existing extracted HD64180 runtime for the
  opcode families directly implicated by the T020 blocker:
  - `DJNZ` (`0x10`)
  - `JR NC,e` (`0x30`) and `JR C,e` (`0x38`)
  - `ADD A,r` / `ADD A,(HL)` (`0x80-0x87`) so the confirmed `ADD A,A`
    blocker is not patched as a one-off
  - `ADD HL,ss` (`0x09`, `0x19`, `0x29`, `0x39`) so both confirmed and
    adjacent 16-bit row/table-stride forms are closed together
  - `SBC HL,ss` (`ED 42`, `ED 52`, `ED 62`, `ED 72`) so the confirmed
    `SBC HL,DE` blocker is covered via the natural handler family
- Adapter T-state timing entries for every opcode added in this milestone,
  matching documented Z80/HD64180 cycle counts.
- Tests and docs needed to lock the newly covered timed opcode surface.
- Narrow runtime/integration coverage proving the canonical PacManV8 T020
  minimal repro can run through the intermission review window without a
  timed-opcode abort.

Explicit non-goals:
- Broad CPU opcode completion beyond the five families above.
- New `ADC`/`SUB`/`SBC A,*` arithmetic families, index-register (`DD`/`FD`)
  work, `CB`-prefixed rotation/bit work, or new block instructions.
- VDP, renderer, scheduler, audio, frontend, debugger, or ROM-side changes.
- Rewriting PacManV8 assembly to avoid the missing opcodes; this milestone is
  explicitly emulator-side closure.
- Opening a general compatibility audit. Any further opcode gaps surfaced after
  the T020 repro clears belong to a later narrow milestone.

Allowed paths:
- `src/core/`
- `third_party/z180/`

Required tests:
- Focused timed CPU coverage for `DJNZ` pinning:
  - `B <- B - 1`
  - taken vs not-taken PC behavior
  - 13/8 T-states for taken/not-taken paths
  - flags left untouched
- Focused timed CPU coverage for `JR NC,e` and `JR C,e` pinning:
  - carry-flag gated taken/not-taken relative control flow
  - 12/7 T-states for taken/not-taken paths
  - flags left untouched
- Focused timed CPU coverage for representative `ADD A,r` / `ADD A,(HL)`
  cases, including the confirmed `ADD A,A` form, pinning:
  - correct `A` result
  - S/Z/H/PV/N/C flag behavior for 8-bit addition
  - 4 T-states for register forms and 7 T-states for `(HL)`
- Focused timed CPU coverage for `ADD HL,ss` pinning:
  - at minimum `ADD HL,BC` and `ADD HL,DE`
  - 16-bit result, half-carry, carry, and `N=0`
  - S/Z/PV preservation
  - 11 T-states
- Focused timed CPU coverage for `SBC HL,ss` pinning:
  - at minimum the confirmed `SBC HL,DE` form
  - 16-bit subtract-with-carry result
  - S/Z/H/PV/N/C flag behavior on borrow / zero-result cases
  - 15 T-states
- Runtime/integration coverage that rebuilds the PacManV8 ROM and proves the
  canonical T020 minimal repro
  `cmake-build-debug/src/vanguard8_headless --rom ... --frames 700 --hash-frame 636`
  completes without a timed-opcode abort, with stable digest output across
  repeat runs.

Exit criteria:
- The emulator no longer aborts on the documented PacManV8 T020 path because
  timed execution is missing any of:
  - `ED 52` (`SBC HL,DE`) at `PC 0x089E`
  - `0x10` (`DJNZ`) at `PC 0x32D3`
  - `0x87` (`ADD A,A`) at `PC 0x32E9`
  - `0x30` (`JR NC,e`) at `PC 0x2F45`
  - `0x09` (`ADD HL,BC`) at `PC 0x333F`
  - or the sister `0x38` / `0x19` forms identified by the same blocker
- The added timed opcode coverage is narrow, documented, and regression-covered.
- PacManV8 T020 becomes actionable again from the emulator side; the actual
  cutscene evidence refresh and acceptance still happen in the PacManV8 repo.

- Bundling unrelated PacManV8 compatibility fixes under the same task.
- Loosening existing timed CPU or integration tests instead of adding the
