# Vanguard 8 — I/O and Timing Reference

## Controller Ports

The Vanguard 8 provides two controller ports on the front of the unit. Each
port is an 8-bit parallel input read via a single I/O port. Buttons are
**active low** (0 = pressed, 1 = released).

### Port Addresses

| Port  | Function                  |
|-------|---------------------------|
| 0x00  | Controller 1 read (P1)    |
| 0x01  | Controller 2 read (P2)    |

### Button Bit Map

```
Bit 7   — D-pad Up
Bit 6   — D-pad Down
Bit 5   — D-pad Left
Bit 4   — D-pad Right
Bit 3   — Button A
Bit 2   — Button B
Bit 1   — Select
Bit 0   — Start
```

### 4-Player Expansion

A 4-player adapter is a planned expansion accessory. Its electrical interface
(additional I/O port, multiplexer select signal, connector pinout) is not yet
fully specified and will be defined in a future revision. The two-controller
interface defined above is the complete, specified controller specification for
the initial hardware revision.

## Timing Reference

### Master Clock

All timing on the Vanguard 8 derives from a single **14.31818 MHz** crystal.

```
Master clock:     14.31818 MHz
CPU clock:        7.15909 MHz  (master ÷ 2)
NTSC subcarrier:  3.57954 MHz  (master ÷ 4)
```

### NTSC Frame Timing

```
Horizontal lines total:    262 per frame
Active scanlines:          212  (V9938 Graphic 4, LN=1)
V-blank scanlines:         50
H-sync frequency:          ~15.734 kHz
Frame rate:                59.94 Hz

CPU cycles per frame:      ~119,304  (at 7.15909 MHz)
CPU cycles per scanline:   ~455
```

### V-Blank

V-blank begins after the last active scanline (line 212) and lasts for
approximately 50 scanlines (~3.2 ms at 59.94 Hz). This is the primary window
for VRAM updates that must not occur during active display:

- Uploading tile patterns to VRAM via VDP ports or HMMC command
- Updating the Sprite Attribute Table and Sprite Color Table
- Updating palette entries
- Queuing multi-tile HMMM blit commands

V-blank is detected via the **VB flag in VDP-A S#0 bit 7**, asserted via
INT0. Enable with R#1 bit 5 on VDP-A.

### H-Blank (Scanline Interrupt)

VDP-A can interrupt at a configurable scanline number. The interrupt is used
for per-line scroll register updates (parallax effects) and mid-frame palette
changes. Vertical scroll via R#23 is fully specified. **Horizontal scroll
(R#26/R#27) behavior in Graphic 4 must be verified against the V9938 Technical
Data Book before relying on per-line horizontal parallax.**

```
Enable:          Set R#0 bit 4 on VDP-A
Target line:     Write scanline number to R#19 on VDP-A
Signal:          VDP-A /INT → CPU /INT0
```

To chain multiple scanline events per frame, update R#19 inside the INT0
handler before returning.

## Interrupt System

### Wiring

```
VDP-A /INT  ────┐
                ├──[4.7 kΩ pull-up to +5V]──▶  CPU /INT0
YM2151 /IRQ ────┘  (open-collector wire-OR)

MSM5205 /VCLK  ────────────────────────────▶  CPU /INT1

(INT2 unconnected, available for cartridge expansion)
```

VDP-A's /INT and YM2151's /IRQ are both **open-collector** outputs. They share
the INT0 line via a wire-OR: either device pulling the line low asserts the
interrupt. A single **4.7 kΩ** pull-up resistor to +5V on the INT0 line is the
only external component required. VDP-B's /INT is not connected to the CPU.
There is no NMI source on the Vanguard 8.

### Interrupt Priority and Sources

```
Line    Source(s)                  Vector           Notes
────────────────────────────────────────────────────────────────────────────────────
INT0    VDP-A V-blank              0x0038 (IM1)     Maskable; check S#0 bit 7
        VDP-A H-blank (scanline)   0x0038 (IM1)     Maskable; check S#1 bit 0
        YM2151 Timer A/B IRQ       0x0038 (IM1)     Maskable; check YM2151 status
INT1    MSM5205 /VCLK              Vectored (I/IL)  Always vectored; see INT1 setup below
INT2    (unassigned)               —
```

**INT0** follows the CPU interrupt mode. Vanguard 8 runs in **IM1**; INT0 calls
**0x0038**.

**INT1 and INT2** use a dedicated vectored response that is **mode-independent** —
it is not affected by the IM0/1/2 setting. The same response path is used by
on-chip peripheral interrupts such as the HD64180 programmable reload timers.
The processor forms the vector pointer from:

```
high byte = I
low byte  = (IL & 0xE0) | fixed_code
```

Fixed low-byte codes relevant to Vanguard 8:

```
INT1 = 0x00
INT2 = 0x02
PRT0 = 0x04
PRT1 = 0x06
```

### INT1 Setup — MSM5205 ADPCM

The following registers must be configured before enabling INT1:

```
; Set I register (vector table high byte)
LD   A, 0x80
LD   I, A           ; I = 0x80 → vector table in SRAM at 0x80xx

; Set IL register (internal addr 0x33)
; Bits 7:5 select the 32-byte vectored-interrupt subtable.
LD   A, 0xE0
OUT0 (0x33), A      ; INT1=0x80E0, INT2=0x80E2, PRT0=0x80E4, PRT1=0x80E6

; Enable INT1 via ITC register (internal addr 0x34)
LD   A, 0x02
OUT0 (0x34), A      ; ITE1 = 1 (bit 1); INT1 enabled

; Store ADPCM handler address in vector table (little-endian)
LD   HL, ADPCM_HANDLER
LD   (0x80E0), HL   ; INT1 vector entry at SRAM 0x80E0–0x80E1
```

INT2 is unassigned; its vector pointer (0x80E2–0x80E3 per the IL setup above)
need not be populated.

### Internal Timer Interrupt Setup — PRT0 / PRT1

The programmable reload timers are not external `INT0` sources. They use the
same internal vectored-interrupt table as `INT1`/`INT2`.

With `I = 0x80` and `IL = 0xE0`, the timer vector words are:

```
PRT0 handler address word at 0x80E4–0x80E5
PRT1 handler address word at 0x80E6–0x80E7
```

Timer interrupts are enabled through the timer control register `TCR`:

```
bit 5 = TIE1   ; enable PRT1 interrupt request
bit 4 = TIE0   ; enable PRT0 interrupt request
bit 1 = TDE1   ; start/stop PRT1 down counting
bit 0 = TDE0   ; start/stop PRT0 down counting
```

When `TMDRn` reaches `0x0000`, `TIFn` becomes `1`. If `TIEn = 1` and CPU
interrupts are enabled, the HD64180 performs the internal vectored interrupt
response for that timer channel.

### INT0 Handler — Source Identification

Reading a status register both retrieves the flag and clears the interrupt
source. The handler must service all pending sources before returning.

**Reading S#0 (V-blank, default register):**
```
IN A, (0x81)    ; Reads S#0, clears VB flag
                ; bit 7 = VB (V-blank), bit 5 = C (sprite collision)
```

**Reading S#1 (H-blank flag):**
```
OUT (0x81), 0x01   ; \ Write R#15 = 1
OUT (0x81), 0x8F   ; /  (selects S#1 for next read)
IN  A, (0x81)      ;   Read S#1; bit 0 = FH (H-blank flag)
OUT (0x81), 0x00   ; \ Restore R#15 = 0
OUT (0x81), 0x8F   ; /
```

**Reading YM2151 status:**
```
IN A, (0x40)    ; bit 1 = Timer A overflow, bit 0 = Timer B overflow
```

### INT1 Handler — ADPCM Nibble Feed

INT1 fires at the MSM5205 sample rate (4, 6, or 8 kHz depending on port 0x60
configuration). The handler must write exactly one 4-bit nibble to port 0x61.

```
; Minimal INT1 handler structure
INT1:
    PUSH AF
    ; fetch next nibble from sample pointer in SRAM, write to 0x61
    POP AF
    EI
    RETI
```

At 8 kHz, INT1 fires every ~894 CPU cycles — a lightweight interrupt that
leaves the CPU with ample time for game logic.

## ROM Banking

ROM banking is performed by writing to the HD64180 internal BBR register
using the `OUT0` instruction. No external latch or mapper chip is required.

```
To switch to bank N (N = 0 to 58):
    BBR value = (N + 1) × 4
    OUT0 (0x39), BBR_value

Bank 0:   BBR = 0x04  →  ROM physical 0x04000–0x07FFF
Bank 1:   BBR = 0x08  →  ROM physical 0x08000–0x0BFFF
...
Bank 58:  BBR = 0xEC  →  ROM physical 0xEC000–0xEFFFF

WARNING: BBR ≥ 0xF0 maps into SRAM. Do not use.
```

The bank window occupies logical **0x4000–0x7FFF**. Code running from this
window must not switch banks mid-execution. Code in 0x0000–0x3FFF (fixed ROM)
and 0x8000–0xFFFF (SRAM) is always accessible regardless of bank state.

## Power and Reset

- **Supply:** 5 V DC, ~1.5 A typical (2 A peak)
- **Reset:** RC + supervisor IC (e.g., MAX690) drives /RESET to all chips
- **Power-on state:** VDP registers cleared, audio silenced, CPU fetches at 0x0000

On reset, HD64180 MMU defaults to CBAR=0xFF, CBR=0, BBR=0. Startup ROM at
0x0000 immediately reconfigures MMU to the Vanguard 8 standard layout
(CBAR=0x48, CBR=0xF0, BBR=0x04).
