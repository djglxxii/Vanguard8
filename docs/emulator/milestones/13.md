# Milestone 13 — Critical Audit Closure for Real ROM Compatibility

Objective:
- Close the highest-value core gaps from the pre-GUI audit that are most likely
  to block real ROM and showcase-ROM testing once the playable desktop
  frontend exists.

Required docs:
- `docs/spec/01-cpu.md`
- `docs/spec/02-video.md`
- `docs/spec/04-io.md`
- `docs/emulator/04-video.md`
- `docs/emulator/07-implementation-plan.md`
- `docs/emulator/11-pre-gui-audit.md`

Allowed scope:
- V9938 VRAM read-ahead behavior, VDP screen-disable behavior, sprite
  compatibility expansion, Graphic 1/2 rendering, and targeted HD64180
  instruction/behavior audit coverage needed for real ROM bring-up.

Explicit non-goals:
- Graphic 5 palette mapping, Graphic 7 direct-color encoding, Text 2 VRAM
  layout decisions, horizontal scroll, or any other still-unverified spec gap.
- Save-state/rewind UX, desktop workflow polish, or full debugger UI work.

Allowed paths:
- `src/core/`
- `tests/`
- `docs/`

Required tests:
- VRAM read-ahead latch coverage.
- Screen-disable coverage.
- Sprite tests for 16x16 size, magnification, and register-relative table
  addressing.
- Graphic 1/2 rendering tests with Sprite Mode 1 coverage.
- Targeted HD64180 tests for the newly audited covered cases.

Exit criteria:
- The critical audit items most likely to block first-wave ROM testing are
  implemented and regression-covered.
- Remaining compatibility gaps are either lower priority or explicitly
  spec-gated.

Forbidden extras:
- Inventing behavior for spec-gated video modes or unresolved scroll details.
- Desktop debugger or UI work that does not directly support the covered core
  compatibility fixes.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
