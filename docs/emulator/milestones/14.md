# Milestone 14 — Lightweight Bring-Up Tooling and Selected Closure

Objective:
- Add only the smaller non-essential items that materially help ROM bring-up
  and debugging, while keeping full heavy debugger/UI work out of scope unless
  it is explicitly promoted later.

Required docs:
- `docs/emulator/00-overview.md`
- `docs/emulator/06-debugger.md`
- `docs/emulator/07-implementation-plan.md`
- `docs/emulator/11-pre-gui-audit.md`

Allowed scope:
- Trace-to-file, symbol loading/annotation, lightweight runtime status output,
  and only those additional renderer/tooling closures that the written spec
  already supports and target ROM bring-up proves necessary.

Explicit non-goals:
- Dear ImGui shell integration or a full live desktop debugger UI.
- Quicksave/quickload, rewind, screenshot, WAV export, auto-resume, or other
  workflow polish that is not required for ROM bring-up.
- Spec-gated video-mode work such as Graphic 5 palette mapping, Graphic 7
  direct-color encoding, or Text 2 layout guesses.

Allowed paths:
- `src/core/`
- `src/debugger/`
- `src/frontend/`
- `tests/`
- `docs/`

Required tests:
- Trace file writer coverage.
- Symbol load/lookup/annotation coverage.
- Mode-specific tests for any renderer added in this milestone.
- Documentation-backed regression notes for the remaining deferred items.

Exit criteria:
- A developer can run a target ROM, see/hear it in the desktop app, and get
  lightweight diagnostic output without a full live debugger UI.
- Remaining omissions are narrow, documented, and intentional.

Forbidden extras:
- Reintroducing a full desktop debugger milestone without an explicit plan
  revision.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
