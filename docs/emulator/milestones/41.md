# Milestone 41 — Timed HD64180 CB-Prefix `SRL H` and `RR L` Coverage for PacManV8 T021

Objective:
- Close the next real-ROM timed CPU compatibility gap exposed after M40
  added the authorized `EX DE,HL` and `OR (HL)` coverage for PacManV8
  T021. The PacManV8 blocked task
  `/home/djglxxii/src/PacManV8/docs/tasks/blocked/T021-pattern-replay-and-fidelity-testing.md`
  now reports that the canonical headless runtime runs past the previous
  M40 opcode gaps and aborts with:

  ```text
  terminate called after throwing an instance of 'std::runtime_error'
    what():  Unsupported timed Z180 opcode 0xCB 0x3C at PC 0x1302
  ```

- `PC=0x1302` is inside `collision_tile_info` (`collision_tile_info`
  begins at `0x124B` per `build/pacman.sym`; the actual failure site is
  documented in the PacManV8 blocker near `collision_prepare_tile` at
  `0x12C7`). The failing instruction is standard Z80 `SRL H`
  (`0xCB 0x3C`), used as part of a right-shift sequence that divides the
  `COLLISION_WORK_INDEX` value by eight:

  ```asm
          ld hl, (COLLISION_WORK_INDEX)
          srl h
          rr l
          srl h
          rr l
          srl h
          rr l
  ```

  Rewriting PacManV8 to avoid the instruction would be an out-of-scope
  emulator workaround.

- The blocked task also identifies the immediate same-path follow-on
  gap: `RR L` (`0xCB 0x1D`) appears directly after the confirmed
  `SRL H` site, and the same `srl h -> rr l` pair repeats three times
  in a row. The M39 CB-prefix dispatch handler already routes
  `SRL A` (`CB 3F`) and `BIT 4..7,A` (`CB 67`/`6F`/`77`/`7F`), but no
  other CB sub-opcodes. Closing `RR L` alongside the confirmed `SRL H`
  gap follows the existing family-closure precedent for same-path
  look-ahead (M38, M39, M40) without broadening into a general
  CB-prefix completion pass.

Allowed scope:
- Timed opcode coverage inside the existing extracted HD64180 runtime
  for exactly these CB-prefix sub-opcodes:
  - `SRL H` (`0xCB 0x3C`) — confirmed failing opcode at `PC 0x1302`.
  - `RR L` (`0xCB 0x1D`) — immediate same-path look-ahead opcode in
    the `collision_prepare_tile` divide-by-eight sequence.
- Implementation shape for `SRL H`:
  - Shift `H` right by one logical bit. Old bit 0 of `H` becomes the
    new carry flag; bit 7 of the new `H` is cleared.
  - Update flags from the resulting byte: S from bit 7 (always 0 after
    SRL), Z from zero result, P/V from even parity; clear H and N.
  - Advance `PC` by 2 (CB prefix byte plus sub-opcode).
  - Adapter timing: **8 T-states**, routed through the existing
    `cb_instruction_tstates` helper introduced in M39.
- Implementation shape for `RR L`:
  - Rotate `L` right through the carry flag. Old bit 0 of `L` becomes
    the new carry; the old carry becomes the new bit 7 of `L`.
  - Update flags from the resulting byte: S from bit 7, Z from zero
    result, P/V from even parity; clear H and N; C as described above.
  - Advance `PC` by 2 (CB prefix byte plus sub-opcode).
  - Adapter timing: **8 T-states**, routed through the existing
    `cb_instruction_tstates` helper.
- Extending the existing `op_cb_prefix` dispatch in
  `third_party/z180/z180_core.cpp` to route the two new sub-opcodes
  while preserving the current fail-fast contract for every still-
  unsupported CB sub-opcode.
- Tests and docs needed to lock the newly covered CB-prefix surface.
- Narrow runtime/integration coverage proving the canonical PacManV8
  T021 replay validation command no longer aborts at `PC=0x1302` with
  `0xCB 0x3C` and no longer aborts at the immediately following
  `0xCB 0x1D` site.

Explicit non-goals:
- **No general CPU opcode sweep.** This milestone covers only
  `SRL H` and `RR L`.
- **No broader SRL coverage.** `SRL B` (`CB 38`), `SRL C` (`CB 39`),
  `SRL D` (`CB 3A`), `SRL E` (`CB 3B`), `SRL L` (`CB 3D`), and
  `SRL (HL)` (`CB 3E`) are out of scope unless a later real-ROM path
  proves they are required.
- **No broader rotate coverage.** `RR B` (`CB 18`), `RR C` (`CB 19`),
  `RR D` (`CB 1A`), `RR E` (`CB 1B`), `RR H` (`CB 1C`), `RR A`
  (`CB 1F`), `RR (HL)` (`CB 1E`), the non-through-carry rotates
  (`RLC`, `RRC`, `RL`), `SLA`, `SRA`, and the undocumented `SLL` form
  are all out of scope.
- **No shift/rotate `(HL)` memory-target forms.**
- **No `RES b,r`, `SET b,r`, or `BIT b,r` expansion.** Existing M39
  `BIT 4..7,A` coverage must stay untouched.
- **No index-register CB variants (`DD CB`, `FD CB`).**
- **No interrupt-controller, scheduler, audio, VDP, frontend,
  debugger, save-state, or observability changes.**
- **No PacManV8 ROM edits.** The T021 evidence refresh happens in the
  PacManV8 repo after this milestone lands.
- **No M39 or M40 scope changes.** The existing CB-prefix dispatch,
  `SRL A`, `BIT 4..7,A`, `EX DE,HL`, and `OR (HL)` handlers remain as
  implemented.

Required tests:
- Focused timed CPU coverage in `tests/test_cpu.cpp` for `SRL H`
  (`0xCB 0x3C`) pinning:
  - `H` shifted right by one, bit 7 of new `H` cleared, old bit 0 of
    `H` → C.
  - S cleared (always, since new bit 7 is 0), Z reflects new `H == 0`,
    H cleared, P/V reflects parity of new `H`, N cleared.
  - `L`, other registers, and `A` are unchanged.
  - `PC` advances by 2.
  - Adapter reports 8 T-states.
  - Include at least one zero-result case and one case where the
    pre-shift value has bit 0 set (carry out).
- Focused timed CPU coverage in `tests/test_cpu.cpp` for `RR L`
  (`0xCB 0x1D`) pinning:
  - `L` rotated right through carry: new bit 7 = old carry, new
    bit 0..6 = old bit 1..7, new C = old bit 0.
  - S from new bit 7, Z from new `L == 0`, H cleared, P/V from
    parity of new `L`, N cleared.
  - `H`, other registers, and `A` are unchanged.
  - `PC` advances by 2.
  - Adapter reports 8 T-states.
  - Include cases covering: carry cleared on entry, carry set on
    entry, zero result, sign-bit result.
- A short sequential test mirroring the PacManV8
  `collision_prepare_tile` pattern: load `HL` with a non-trivial
  value, run `SRL H -> RR L` three times in a row, and confirm the
  final `HL` equals `HL_initial >> 3` with the expected carry-out
  captured in the flags.
- A negative guard confirming a nearby out-of-scope CB sub-opcode
  such as `SRL L` (`CB 3D`) or `RR H` (`CB 1C`) still raises the
  existing `Unsupported timed Z180 opcode 0xCB 0x<subop>` runtime
  error pattern.
- Adapter-level T-state coverage confirming both `CB 3C` and
  `CB 1D` are routed through `cb_instruction_tstates` and classified
  as 8 T-states.
- Non-perturbation guard against a fixture ROM that does not exercise
  the new opcodes: frame, audio, and event-log digests must be
  byte-identical before and after the fix.
- Runtime/integration coverage that rebuilds the PacManV8 ROM and
