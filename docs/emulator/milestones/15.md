# Milestone 15 — Timed HD64180 INT1 Audio Bring-Up Patch

Objective:
- Fix the narrow timed-CPU compatibility gap that blocks the first real
  ROM-driven audio scene: vectored `INT1` execution during scheduled runtime
  plus only the additional opcode coverage that documented audio-scene logic
  proves necessary.

Required docs:
- `docs/spec/01-cpu.md`
- `docs/spec/03-audio.md`
- `docs/spec/04-io.md`
- `docs/emulator/03-cpu-and-bus.md`
- `docs/emulator/05-audio.md`
- `docs/emulator/07-implementation-plan.md`
- `showcase/tasks/blocked/SR05-T01-audio-verification-scene-and-checkpoints.md`

Allowed scope:
- Timed extracted-Z180 interrupt-path fixes.
- The smallest timed opcode-surface expansion required by the blocked
  ROM-driven audio path.
- Normal emulator runtime/core test coverage and docs updates needed to pin the
  fix.

Explicit non-goals:
- General full-Z80/Z180 opcode parity work.
- New showcase-ROM scene implementation or showcase-only runtime shortcuts.
- Broad audio-engine redesign outside the documented YM2151/AY-3-8910/MSM5205
  surface already accepted by milestones 7 and 14.

Allowed paths:
- `src/core/`
- `third_party/z180/`
- `tests/`
- `docs/`

Required tests:
- A regression that reproduces the pre-fix timed `INT1` real-ROM failure.
- Runtime/integration coverage proving the blocked audio-window transition can
  execute through the normal frame loop.
- Focused tests for any newly supported timed opcode handlers.

Exit criteria:
- The timed CPU path no longer falls into the vector-table pointer or throws an
  unsupported-opcode exception on the first blocked real-ROM audio window.
- The newly added opcode/interrupt behavior is regression-covered and narrow.
- The showcase milestone-5 blocker can be revisited without needing an
  emulator-side plan change first.

Forbidden extras:
- Turning the milestone into a general CPU completion pass.
- Reworking unrelated frontend, debugger, or showcase-ROM logic.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
