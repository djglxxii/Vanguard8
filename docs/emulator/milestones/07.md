# Milestone 7 — Audio Chips, Mixer, and Dedicated ADPCM Interrupt Flow

Objective:
- Add the documented audio surface and its CPU timing relationships.

Required docs:
- `docs/spec/00-overview.md`
- `docs/spec/01-cpu.md`
- `docs/spec/03-audio.md`
- `docs/emulator/05-audio.md`
- `docs/emulator/07-implementation-plan.md`

Allowed scope:
- YM2151, AY-3-8910, MSM5205, mixer/resampler, and INT0/INT1 audio timing.

Explicit non-goals:
- DMA/timers, debugger panels, save states, or replay tooling.

Allowed paths:
- `src/core/audio/`
- `src/core/`
- `third_party/nuked-opm/`
- `third_party/ayumi/`
- `third_party/msm5205/`
- `tests/`

Required tests:
- YM2151 status/busy behavior.
- AY latch/read/write behavior.
- MSM5205 control and `/VCLK` cadence updates.
- INT1 timing stability and deterministic headless audio hashes.

Exit criteria:
- Audio output is synchronized and deterministic in core/headless execution.

Implementation note:
- Milestone 7 is locked away from `src/frontend/`, so deterministic audio
  verification is satisfied by the core mixer/resampler path and its exposed
  hashes rather than by a live SDL audio callback.

Forbidden extras:
- Frontend-only audio shortcuts that bypass core timing.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
