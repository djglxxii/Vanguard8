# Milestone 2 — CPU Core, MMU, Interrupt Lines, and Boot Sequence

Objective:
- Integrate the Z180/HD64180 core through an adapter and prove boot-time MMU
  behavior against the Vanguard 8 mapping.

Required docs:
- `docs/spec/00-overview.md`
- `docs/spec/01-cpu.md`
- `docs/emulator/03-cpu-and-bus.md`
- `docs/emulator/07-implementation-plan.md`

Allowed scope:
- CPU adapter, reset behavior, MMU translation, INT0/INT1 plumbing, and boot
  ROM verification paths.

Explicit non-goals:
- Scheduler, video rendering, controller input, and audio implementation.

Allowed paths:
- `src/core/`
- `third_party/z180/`
- `tests/`

Required tests:
- MMU boot mapping and bank-window translation.
- `OUT0` writes to internal MMU registers.
- INT0 IM1 behavior and INT1 vectored behavior.
- Illegal `BBR >= 0xF0` warning behavior.

Exit criteria:
- A test ROM can boot, switch ROM banks, and read/write SRAM correctly.

Forbidden extras:
- DMA, timers, or debugger work from later milestones.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
