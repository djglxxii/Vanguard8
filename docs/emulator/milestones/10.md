# Milestone 10 — Save States, Rewind, Replay, and Headless CI

Objective:
- Add deterministic regression protection before compatibility expansion.

Required docs:
- `docs/emulator/00-overview.md`
- `docs/emulator/02-emulation-loop.md`
- `docs/emulator/06-debugger.md`
- `docs/emulator/07-implementation-plan.md`

Allowed scope:
- Save-state serialization, rewind, replay, headless hashing, and replay-based
  `ctest` integration.

Explicit non-goals:
- New emulator hardware features outside state capture and regression tooling.

Allowed paths:
- `src/core/`
- `src/frontend/headless.*`
- `tests/`

Required tests:
- Save/load round-trip coverage for CPU, MMU, VRAM, palettes, audio, and
  scheduler state.
- Stable replay frame/audio hashes.
- Rewind restores prior states without AV desynchronization.

Exit criteria:
- CI can catch regressions with deterministic replay assets.

Forbidden extras:
- Compatibility fixes unrelated to regression tooling.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
