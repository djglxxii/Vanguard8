# Milestone 8 — HD64180 DMA/Timers and VDP Command Engine

Objective:
- Implement the internal CPU peripherals and VDP acceleration path software
  will depend on.

Required docs:
- `docs/spec/01-cpu.md`
- `docs/spec/02-video.md`
- `docs/emulator/03-cpu-and-bus.md`
- `docs/emulator/04-video.md`
- `docs/emulator/07-implementation-plan.md`

Allowed scope:
- DMA channels 0 and 1, timers/counters, VDP command execution, and `S#2.CE`
  timing.

Explicit non-goals:
- Debugger, replay, or compatibility-polish work.

Allowed paths:
- `src/core/cpu/`
- `src/core/video/`
- `src/core/`
- `tests/`

Required tests:
- DMA source/destination rules.
- DMA-to-I/O port dispatch reuse.
- Timer behavior stability.
- `CE` in-progress and completion timing.

Exit criteria:
- ROMs can use DMA, timers, and VDP blits without emulator-only shortcuts.

Forbidden extras:
- Unspecified VDP behavior outside the documented command set.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
