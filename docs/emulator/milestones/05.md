# Milestone 5 — Dual VDPs, Compositing, Sprites, and VDP-A Interrupt Semantics

Objective:
- Reach the first Vanguard 8-specific video milestone with dual-VDP
  compositing and correct CPU-visible interrupt behavior.

Required docs:
- `docs/spec/00-overview.md`
- `docs/spec/02-video.md`
- `docs/emulator/04-video.md`
- `docs/emulator/07-implementation-plan.md`

Allowed scope:
- Second VDP instance, compositor, Graphic 4/3 sprite path, VDP-A interrupt
  semantics, VDP-B interrupt isolation, and layer toggles.

Explicit non-goals:
- Audio, controller UX, or debugger panels.

Allowed paths:
- `src/core/video/`
- `src/frontend/`
- `tests/`

Required tests:
- VDP-A color `0` transparency and nonzero override behavior.
- VDP-B interrupt isolation.
- `S#0` and `S#1` flag clear behavior on VDP-A.
- Covered sprite collision and overflow cases.

Exit criteria:
- A composited frame renders correctly and VDP-A IRQs reach INT0.

Forbidden extras:
- Any VDP-B to CPU interrupt path.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
