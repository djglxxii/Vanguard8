# Milestone 40 — Timed HD64180 EX DE,HL and OR (HL) Coverage for PacManV8 T021

Objective:
- Close the next real-ROM timed CPU compatibility gap exposed after M39
  added the authorized 16-bit pair INC/DEC and narrow CB-prefix surface
  for PacManV8 T021. The PacManV8 blocked task
  `/home/djglxxii/src/PacManV8/docs/tasks/blocked/T021-pattern-replay-and-fidelity-testing.md`
  now reports that the canonical headless runtime runs past the previous
  M39 opcode gaps and aborts with:

  ```text
  terminate called after throwing an instance of 'std::runtime_error'
    what():  Unsupported timed Z180 opcode 0xEB at PC 0x11DA
  ```

- `PC=0x11DA` is inside `collision_init`. The failing instruction is
  standard Z80 `EX DE,HL` (`0xEB`), used to swap the current cell pointer
  with the loop state while setting the pellet-bitset mask. Rewriting
  PacManV8 to avoid the instruction would be an out-of-scope emulator
  workaround.

- The blocked task also identifies the immediate same-path follow-on gap:
  `OR (HL)` (`0xB6`) appears directly after the confirmed `EX DE,HL`
  site in `collision_init`. The timed dispatch table already covers the
  register `OR` forms `0xB0..0xB5` and `0xB7`, plus immediate `OR n`
  from M25 and `OR E` from M28, but not the `(HL)` memory operand form.
  Closing `OR (HL)` with the confirmed `EX DE,HL` gap follows the
  existing family-closure precedent for same-path look-ahead without
  broadening into a general opcode-completion pass.

Allowed scope:
- Timed opcode coverage inside the existing extracted HD64180 runtime
  for exactly these opcodes:
  - `EX DE,HL` (`0xEB`) — confirmed failing opcode at `PC 0x11DA`.
  - `OR (HL)` (`0xB6`) — immediate same-path look-ahead opcode in
    `collision_init`.
- Implementation shape for `EX DE,HL`:
  - Swap the full 16-bit `DE` and `HL` register pairs.
  - Advance `PC` by 1.
  - Leave all flags (S/Z/H/PV/N/C) untouched.
  - Adapter timing: **4 T-states**.
- Implementation shape for `OR (HL)`:
  - Read one byte from logical address `HL`.
  - Compute `A <- A | (HL)`.
  - Set S from result bit 7, Z from zero result, P/V from even parity;
    clear H, N, and C.
  - Advance `PC` by 1.
  - Adapter timing: **7 T-states**.
- Tests and docs needed to lock the newly covered timed opcode surface.
- Narrow runtime/integration coverage proving the canonical PacManV8
  T021 replay validation command no longer aborts at `PC=0x11DA` with
  `0xEB` and no longer aborts at the immediately following `0xB6` site.

Explicit non-goals:
- **No general CPU opcode sweep.** This milestone covers only
  `EX DE,HL` and `OR (HL)`.
- **No broader exchange-family coverage.** `EX AF,AF'` (`0x08`),
  `EXX` (`0xD9`), `EX (SP),HL` (`0xE3`), and index-register exchange
  forms are out of scope unless a later real-ROM path proves they are
  required.
- **No broader logical-OR expansion.** Register `OR` forms already
  covered elsewhere must not be rewritten, and no `OR IX/IY` indexed
  variants or unrelated arithmetic/logical families are included.
- **No interrupt-controller, scheduler, audio, VDP, frontend,
  debugger, save-state, or observability changes.**
- **No PacManV8 ROM edits.** The T021 evidence refresh happens in the
  PacManV8 repo after this milestone lands.
- **No M39 scope changes.** The M39 pair INC/DEC and CB-prefix handlers
  remain as implemented; this milestone does not reopen that contract.

  - `PC` advances by 1.
  - All flags are preserved.
  - Other registers are unchanged.
  - Adapter reports 4 T-states.
- Focused timed CPU coverage in `tests/test_cpu.cpp` for `OR (HL)`

  - The operand is read from logical address `HL`.
  - `A` becomes `A | (HL)`.
  - S, Z, and P/V match the result; H, N, and C clear.
