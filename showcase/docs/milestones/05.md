# Showcase Milestone 5 — Audio Verification Scene and Audio Checkpoints

Objective:
- Add the audio verification scene that isolates YM2151, AY-3-8910, MSM5205,
  and mixed playback in a deterministic loop.

Required docs:
- `docs/spec/03-audio.md`
- `docs/spec/04-io.md`
- `showcase/docs/showcase-rom-plan.md`
- `showcase/docs/showcase-implementation-plan.md`

Allowed scope:
- Audio scene logic, deterministic sequence/sample source data, generated audio
  inputs, and audio checkpoint documentation.

Explicit non-goals:
- Final system-validation scene or milestone-freeze documentation.
- Audio content polish beyond what is required to hear and hash each chip path.

Allowed paths:
- `showcase/`

Required verification:
- The audio scene presents YM-only, AY-only, MSM-only, and mixed windows.
- A documented headless run can emit a stable audio hash for review.
- Trace notes identify the INT1-driven MSM feed window.

Exit criteria:
- A reviewer can run the documented command, inspect the audio hash, and map
  the captured windows back to the intended chip sections.

Forbidden extras:
- Any audio pipeline work that depends on emulator changes outside the showcase
  workspace.

Verification commands:
```bash
python3 showcase/tools/package/build_showcase.py
python3 showcase/tools/package/run_showcase_headless.py --frames 1200 --trace build/showcase/m5.trace --hash-audio --symbols build/showcase/showcase.sym
```
