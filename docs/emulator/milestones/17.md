# Milestone 17 — Full-Range VDP CPU VRAM Addressing Compatibility

Objective:
- Complete the covered V9938 CPU-visible VRAM address path across the full
  64 KB VRAM space so high-address `Graphic 4` and `Graphic 6` content written
  through the data/control ports no longer wraps into the low address window.

Required docs:
- `docs/spec/02-video.md`
- `docs/spec/04-io.md`
- `docs/emulator/04-video.md`
- `docs/emulator/07-implementation-plan.md`
- `docs/emulator/08-compatibility-audit.md`

Allowed scope:
- VDP data/control-port addressing behavior in the existing `src/core/video/`
  path
- Tests and docs needed to lock the full 64 KB CPU VRAM-addressing surface
- Narrow runtime/integration coverage for the already-planned mixed-mode HUD
  content that depends on high-address `Graphic 6` data

Explicit non-goals:
- New VDP modes or broader mode-completion work
- Command-engine expansion beyond the already covered surface
- Frontend/headless tooling changes beyond what a direct runtime regression
  needs
- New showcase-ROM scenes or unrelated ROM polish

Allowed paths:
- `src/core/`
- `tests/`
- `docs/`

Required tests:
- Direct VDP CPU write coverage for addresses above `0x3FFF`
- Direct VDP CPU read coverage for the same high-address path
- Integration/runtime coverage proving high-address `Graphic 6` HUD content no
  longer aliases into low VRAM

Exit criteria:
- CPU-driven VRAM access can reach the full documented 64 KB space without
  upper-window aliasing
- Covered ROM/runtime paths that rely on high-address `Graphic 6` content
  render deterministically through the existing compositor
- Remaining VDP feature gaps stay documented rather than partially implemented

Forbidden extras:
- Turning the milestone into a general V9938 rewrite
- Bundling unrelated frontend/debugger/tooling work into the same milestone
- Expanding unsupported mode-specific command behavior under the same task

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
