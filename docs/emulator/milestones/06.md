# Milestone 6 — Input, ROM Workflow, and Minimal User-Facing Frontend

Objective:
- Make the emulator usable for ROM loading and two-player control while
  keeping scope tight.

Implementation note:
- In the current repo, milestone 6 may satisfy the SDL-facing input and ROM
  workflow requirements with deterministic frontend abstractions that use the
  documented SDL-style binding names and route into the core controller ports,
  even if a full live SDL window/event loop is still minimal.
- Milestone 6 includes the narrow core plumbing needed to make the frontend
  usable: controller ports on `Bus`, config parsing/writing, and ROM attachment
  in `Emulator`.
- In the current repo, ROM loading is limited to raw cartridge images validated
  against the documented Vanguard 8 cartridge constraints (16 KB pages, maximum
  960 KB). "Invalid format" for milestone 6 means a file that cannot satisfy
  those cartridge rules.

Required docs:
- `docs/spec/00-overview.md`
- `docs/spec/04-io.md`
- `docs/emulator/01-build.md`
- `docs/emulator/07-implementation-plan.md`

Allowed scope:
- SDL keyboard/gamepad mapping, active-low controller decoding, config
  creation/loading, ROM open flow, recent list, fullscreen, scale modes, and
  frame pacing.

Explicit non-goals:
- Audio chip integration, debugger tooling, or advanced frontend polish.

Allowed paths:
- `src/core/io/`
- `src/core/bus.*`
- `src/core/config.*`
- `src/core/emulator.*`
- `src/core/CMakeLists.txt`
- `src/frontend/`
- `tests/`

Required tests:
- Active-low controller reads.
- Independent player 1 and player 2 ports.
- Oversized and invalid ROM rejection.

Exit criteria:
- A user can load a ROM, see video, and provide controller input.

Forbidden extras:
- 4-player expansion or unspecified input hardware.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
