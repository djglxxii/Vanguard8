# Milestone 3 — Master-Cycle Scheduler and Main Emulation Loop

Objective:
- Establish deterministic event scheduling and the frame loop shape the
  remaining subsystems depend on.

Required docs:
- `docs/spec/00-overview.md`
- `docs/spec/01-cpu.md`
- `docs/spec/03-audio.md`
- `docs/emulator/02-emulation-loop.md`
- `docs/emulator/07-implementation-plan.md`

Allowed scope:
- Master-cycle counter, scheduler, frame accounting, event timing, and
  headless run/pause/frame-step behavior.

Explicit non-goals:
- Visible video output, controller UX, or full audio mixing.

Allowed paths:
- `src/core/`
- `src/frontend/headless.*`
- `tests/`

Required tests:
- Deterministic H-blank, V-blank, V-blank-end, and `/VCLK` event positions.
- Stable INT0/INT1 timing across repeated runs.
- Monotonic CPU cycle accounting.

Exit criteria:
- Headless execution can run for N frames with stable event logs.

Forbidden extras:
- SDL/OpenGL display or debugger UI work.

Verification commands:
```bash
cmake -B build -DCMAKE_TOOLCHAIN_FILE=third_party/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
ctest --test-dir build --output-on-failure
```
