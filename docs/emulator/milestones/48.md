# Milestone 48 — Re-pin PacManV8 T017 300-Frame Audio Digest

Status note, 2026-04-26:
- Milestone 47 closed cleanly: the full MAME HD64180 / Z180 core is
  imported at pinned upstream commit
  `c331217dffc1f8efde2e5f0e162049e39dd8717d`, the per-opcode timing
  tables and `"Unsupported timed Z180 opcode …"` throw sites are
  gone from `src/core/cpu/`, the replay fixture's frame-4 digest
  was unaffected, and the canonical PacManV8 T021 harness passes
  both replay cases end-to-end.
- During M47 verification, the imported core's more accurate audio
  timing produced a uniform digest shift in exactly one ctest entry
  outside M47's allowed paths: the test
  `PacManV8 T017 audio/video output is nonzero after instruction-
  granular audio timing` in `tests/test_frontend_backends.cpp`
  pins a 300-frame PacManV8 audio SHA-256 at line 387. The pinned
  hash is the M33-era pre-import value
  `61ca417ef206a0762ea3691cb0e48f5bf567205beffed27877d27afca839e7cc`;
  the post-import value is
  `a765959a62e9a5afe9d075206efb8943e3141ef4274844a07c35f21c20d1ab27`,
  reproducible byte-identical across three repeat runs.
- The structural assertions in the same test still pass post-import:
  the audio buffer has nonzero bytes, the composed frame has nonzero
  bytes, and `emulator.cpu().pc()` is not `0x2B8B` (the recovery-
  detection PC). Only the pinned digest needs updating; this is the
  exact "more accurate timing → digest shift" outcome M47
  anticipated.
- `tests/test_frontend_backends.cpp` is not in M47's allowed paths
  (M47 only allowed digest re-pinning inside `tests/replays/`). The
  re-pin is therefore deferred to this milestone. M48 is narrow on
  purpose: it is a single-line digest re-pin plus its evidence and
  doc closure. No other test, source, or spec change is in scope.

Objective:
