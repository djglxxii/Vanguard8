# Milestone 42 — Timed HD64180 `SCF` Coverage for PacManV8 T021

Objective:
- Close the next real-ROM timed CPU compatibility gap exposed after
  M41 added the authorized `SRL H` and `RR L` CB-prefix coverage for
  PacManV8 T021. The PacManV8 blocked task
  `/home/djglxxii/src/PacManV8/docs/tasks/blocked/T021-pattern-replay-and-fidelity-testing.md`
  now reports that the canonical headless runtime runs past the
  previous M41 opcode gaps and aborts with:

  ```text
  terminate called after throwing an instance of 'std::runtime_error'
    what():  Unsupported timed Z180 opcode 0x37 at PC 0x1315
  ```

- `PC=0x1315` is immediately after the `collision_prepare_tile`
  divide-by-eight sequence closed by M41 and a follow-on `LD A,(nn)`
  read from `COLLISION_WORK_BASE`. ROM bytes around the failure:

  ```text
  00001310: 81 19 3a 13 82 37 c9 b7 c9 21 20 81 cd 38 13 b7
                           ^^
                           SCF at PC=0x1315
  ```

  The failing instruction is standard Z80 `SCF` (`0x37`) used as the
  "return with carry set" idiom — `SCF` followed immediately by `RET`
  (`0xC9`, already timed) — so the caller can branch on the
  collision-check outcome. Rewriting PacManV8 to avoid `SCF` would be
  an out-of-scope emulator workaround.

- The immediate downstream path (`RET` / `OR A` / `RET` / `LD HL,nn`
  / `CALL nn` / `OR A` / `RET NZ` / ...) is already fully timed, so
  no same-path look-ahead opcode is bundled into this milestone. M42
  is therefore narrower than M39/M40/M41 by design: a single opcode
  with tight flag semantics.

Allowed scope:
- Timed opcode coverage inside the existing extracted HD64180 runtime
  for exactly one opcode:
  - `SCF` (`0x37`) — confirmed failing opcode at `PC 0x1315`.
- Implementation shape for `SCF`:
  - Set the carry flag (C = 1).
  - Clear the half-carry flag (H = 0) and the add/subtract flag
    (N = 0).
  - Leave the sign, zero, and parity/overflow flags (S, Z, P/V) and
    all registers other than F untouched.
  - Advance `PC` by 1.
  - Adapter timing: **4 T-states**.
- Tests and docs needed to lock the newly covered timed opcode
  surface.
- Narrow runtime/integration coverage proving the canonical PacManV8
  T021 replay validation command no longer aborts at `PC=0x1315`
  with `0x37`.

Explicit non-goals:
- **No general CPU opcode sweep.** This milestone covers only `SCF`.
- **No `CCF` coverage.** `CCF` (`0x3F`) is the natural sister form,
  but it is not on any confirmed T021 active path and must stay out
  of scope until a later real-ROM path proves it is required.
- **No broader flag-manipulation expansion.** No bit/rotate/shift
  additions, no index-register forms, no `DAA`, no `CPL`.
- **No interrupt-controller, scheduler, audio, VDP, frontend,
  debugger, save-state, or observability changes.**
- **No PacManV8 ROM edits.** The T021 evidence refresh happens in
  the PacManV8 repo after this milestone lands.
- **No M39/M40/M41 scope changes.** Those handlers and their adapter
  T-state entries remain untouched.

Required tests:
- Focused timed CPU coverage in `tests/test_cpu.cpp` for `SCF`
  (`0x37`) pinning:
  - Entry with `C = 0`, `H = 1`, `N = 1` and arbitrary S/Z/PV
    pattern: after execution `C = 1`, `H = 0`, `N = 0`, and the
    preserved S/Z/PV bits equal their pre-execution values.
  - Entry with `C = 1`, `H = 0`, `N = 0` and a different S/Z/PV
    pattern: same post-conditions as above (C stays set).
  - `A` and all other registers (`B`, `C`, `D`, `E`, `H`, `L`, `BC'`,
    `DE'`, `HL'`, `IX`, `IY`, `SP`) must be unchanged.
  - `PC` advances by 1.
  - Adapter reports 4 T-states.
- A short sequential test mirroring the PacManV8 `SCF -> RET`
  idiom: set up a return address on the stack, execute `SCF`, then
  `RET`, and confirm `PC` equals the return target with carry set
  and the other preserved flags intact.
- A negative guard confirming a nearby out-of-scope flag-manipulation
  opcode `CCF` (`0x3F`) still raises the existing
  `Unsupported timed Z180 opcode 0x3F at PC` runtime error pattern.
- Adapter-level T-state coverage confirming `0x37` is classified as
  4 T-states.
- Non-perturbation guard against a fixture ROM that does not
  exercise the new opcode: frame, audio, and event-log digests must
  be byte-identical before and after the fix.
- Runtime/integration coverage that rebuilds the PacManV8 ROM and
  proves the canonical T021 command:
  ```bash
  cd /home/djglxxii/src/PacManV8
  python3 tools/build.py
  python3 tools/pattern_replay_tests.py \
    --evidence-dir tests/evidence/T021-pattern-replay-and-fidelity-testing
  ```
  no longer aborts at `PC=0x1315` with `Unsupported timed Z180
  opcode 0x37`, and reaches the next replay-validation checkpoint
  or records the next out-of-scope opcode blocker explicitly.

Exit criteria:
- The emulator no longer aborts on the documented PacManV8 T021
  path with `Unsupported timed Z180 opcode 0x37 at PC 0x1315`.
- The added timed opcode coverage for `SCF` is narrow, documented,
  and regression-covered, with an explicit negative test for the
  nearby out-of-scope `CCF` opcode.
- `ctest --test-dir cmake-build-debug --output-on-failure` passes
  with the new opcode tests included, and no pre-existing CPU test
  is relaxed or removed.
- Deterministic digests for ROMs that do not depend on the new
  opcode are unchanged.
- PacManV8 T021 becomes actionable again from the emulator side;
  the actual replay-checkpoint evidence and acceptance still happen
  in the PacManV8 repo.

- Bundling unrelated PacManV8 compatibility fixes under the same

- Loosening existing timed CPU or integration tests instead of
